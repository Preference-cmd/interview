## 2. 关键 Prompt 记录（至少 5 条）

### Prompt #1: [初始化repo，环境设置]

```
以reference内容为蓝本，进行基础环境配置和预定义的代码导入。
backend使用uv + py3.12 + ruff。
frontend使用next.js + tailwindcss v4 + shadcn + recharts，仅进行初始化
```

**结果**：AI 根据需求文档生成基础项目结构，并配置我选定的技术栈

---

### Prompt #2: [开发文档及记录设置]

```
将CLAUDE.md的内容并入AGENTS.md，并将CLAUDE.md改为指向AGENTS.md的软链。
所有的开发文档放入 doc/，README作为doc/的索引，STATE用于记录开发状态。
使用superpowers workflow，改动后需要更新维护相关文档
```

**结果**：AI 持续维护doc/ ，并且可以跨session恢复。开发工作流严格遵循superpowers workflow

---

### Prompt #3: [前端设计系统]

```
添加前端设计规范，需要完全遵循frontend/DESIGN.md，导入的组件需要注意设计迁移
```

**结果**：AI 在进行前端设计时会直接使用设定好的设计系统，维持设计一致性

---

### Prompt #4: [提示AI逐步重构]

```
我认为编排模块的设计非常臃肿，你的观察是？
```

**结果**：AI给出自己的观察意见，人工提示下一步的改进方向

---

### Prompt #5: [子代理驱动开发]

```
使用subagent执行计划，每步进行验证
```

**结果**：AI 可以安全执行长任务

## 3. AI 错误修正记录（至少 2 条）

### AI 错误 #1: [初始的one shot生成的代码结构过于臃肿]

- **问题**：AI 哪里错了、为什么错
- **发现方式**：通过人工查看代码结构
- **修复**：提示AI进行处理和优化
- **教训**：最初的文档也需要详细一些

---

### AI 错误 #2: [前端样式混乱]

- **问题**：AI 生成的前端界面的layout和样式混乱，可读性差
- **发现方式** 启动前端开发服务器后人工审核
- **修复**：引入了设计系统DESIGN.md
- **教训**：早期就需要引入设计系统以保持一致性

## 5. 验证方法

| 验证方式 | 具体做法 |
|----------|----------|
| 单元测试 + 集成测试 | 每次改动后由AI自动进行验证步骤 |
| AI + 人工测试 | 让AI启动前后端，与AI一起进行测试，并让AI记录接下来的改进方向 |
| 日志检查 | AI自动检查日志输出以发现潜在问题 |

## 6. 总结（可选）

AI 在本项目中的主要作用：
- 优势：可以快速按照指令生成代码，且范围越小，交付的质量越高。在端到端测试方面也比传统工具拥有更大优势
- 局限：根据使用的模型的不同，有不同的局限性，有些模型擅长不同的部分。但总的来说它们都会受到上下文窗口的限制

你的驾驭策略：
- 尽可能地进行模块化和精简代码，以最大程度发挥AI优势和避免在逼近窗口上限时产生的问题
- 让AI自动管理文档和索引，避免每次打开session需要重新载入大量上下文
- 让AI使用subagent生成代码，再让主agent进行evaluation，可以显著提升代码质量和发现潜在问题
