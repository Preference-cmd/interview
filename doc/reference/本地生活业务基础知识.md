# 本地生活业务基础知识

> 面试背景资料，请在笔试前阅读

本文档帮助你理解笔试题干中的业务背景，包括"本地生活"是什么、代运营做什么、以及商家运营的核心流程与关键指标。

---

## 一、什么是"本地生活"业务

本地生活服务（Local Life Services）是指通过互联网平台（如美团、大众点评）将本地线下商户与周边消费者连接起来的 O2O（Online to Offline）服务模式。其核心特征是：

- **消费发生在本地**：用户通过手机发现附近的餐厅、美容、休闲娱乐等商家
- **服务履约在本地**：用户到店消费（到店）或商家配送上门（到家）
- **平台提供流量与工具**：美团/点评等平台提供搜索、推荐、团购、评价、推广等全套工具

### 两大核心板块

| 板块 | 含义 | 典型场景 | 平台工具 |
|------|------|---------|---------|
| **到店业务** | 消费者到店消费，线上下单、线下体验 | 团购套餐、代金券、收藏打卡、到店核销 | 美团/大众点评 团购、秒杀、特价团、推广通 |
| **到家业务** | 商户通过外卖平台将商品配送到消费者手中 | 外卖点餐、即时配送、闪购 | 美团外卖商家版 App、开店宝 |

## 二、什么是"代运营"

很多餐饮商家擅长做菜，但不懂线上运营。代运营公司就是帮助这些商家在美团/点评等平台上完成"开店→装修→营销→评价管理→数据复盘"全链路运营的专业服务方。核心价值在于：帮商家拿到更多曝光、更高转化率、更好的星级评分，最终提升营收。

### 代运营的主要工作内容

| 工作模块 | 具体内容 | 对应平台操作 |
|---------|---------|------------|
| **门店装修** | 设计入口图、套餐图、五连图；完善店铺基础信息（名称、分类、标签、营业时间） | 美团商户后台、开店宝 App |
| **团单管理** | 设计套餐组合、定价策略、代金券设置；维护招牌菜、爆款套餐置顶 | 团单创建/修改、价格调整、活动报名 |
| **营销推广** | 报名秒杀/特价团/立减活动；开通推广通投放；策划收藏打卡活动 | 平台活动报名、推广后台、数据监控 |
| **评价管理** | 引导顾客写优质评价；好评回复、差评申诉处理；维护星级评分 | 评价回复模板、申诉后台、打卡活动 |
| **数据分析** | 周/月数据报表；客流分析、竞对调研；运营策略调整建议 | 后台数据报表、竞品分析工具 |

## 三、商家生命周期与运营流程

一个新商家从入驻到常态运营，通常经历以下阶段。这也是笔试题中"商家生命周期"的业务背景：

| 时间节点 | 阶段 | 核心工作 | 关键节点 |
|---------|------|---------|---------|
| 第1–2天 | 入场沟通 | 拉群建联、收集账号密码、收集品牌素材、电话沟通营业详情 | 必须获取开店宝登录手机号、菜单成本表、门头环境照 |
| 第3–4天 | 方案确认 | 商圈竞对调研、菜单初步确认、制定推广及补单计划、确认活动方案 | 商圈前5名竞品分析、成本表确认、装修风格确认 |
| 第5天 | 门店基建 | 设计出图（logo、海报、商品图）、内部审核后同步商家确认、上传门店 | ROS分基础项完善、团单上架、营销活动报名 |
| 第6天起 | 正式营业 | 开始推广投放、日常巫检（商品上下架、估清、异常反馈）、评价维护 | 新店前一周人工测试推广金额，一周稳定后固定 |
| 长期 | 常态运营 | 周/月数据报表、节假日活动策划、星级提升、持续优化套餐与推广策略 | 周报输出、数据对比、运营策略迭代 |

## 四、关键业务概念词表

以下是笔试题中会频繁出现的业务术语，建议提前熟悉：

| 术语 | 含义说明 |
|------|---------|
| **ROS分** | 店铺经营评分（Restaurant Online Score），美团平台对门店的综合评分，直接影响曝光量和排名。包括基础信息完善度、商品丰富度、活动参与、推广投放、评价等多个维度 |
| **团单/团购** | 商家在美团/点评上发布的优惠套餐，用户线上购买、到店核销。是到店业务的核心商品形态 |
| **代金券** | 用户线上购买优惠券（如"50元抵100元"），到店使用，用于吸引新客、提升客单价 |
| **秒杀/特价团** | 平台提供的限时低价活动，商家参与后可获得额外流量入口和曝光加权 |
| **推广通** | 美团的付费推广工具，商家设置每日预算和时段，系统自动在美食频道进行曝光推荐 |
| **星级评分** | 美团星级和大众点评星级，基于用户评价计算，直接影响用户决策和平台排名。代运营的核心KPI之一 |
| **收藏打卡** | 用户在大众点评上收藏门店并打卡签到，有助于提升门店热度和搜索排名 |
| **商圈/竞对** | 门店周边的商业区域以及同品类竞争对手。运营需要调研商圈内前5名竞品的套餐、价格、活动策略 |
| **开店宝** | 美团商家端 App，用于管理门店、查看订单、操作活动确认、素材检查等日常运营 |
| **GMV** | 商品交易总额（Gross Merchandise Volume），衡量门店线上营收的核心指标 |
| **曝光→访问→转化** | 本地生活的核心漏斗：曝光量（被用户看到）→ 访问量（点击进店）→ 转化量（下单购买）。运营的所有动作都是为了优化这个漏斗 |

## 五、与笔试题的关联说明

笔试题干中描述的"本地生活代运营团队的 Multi-Agent 运营系统"，本质上就是用 AI 自动化来替代上述人工运营工作。以下是四类 Agent 与实际业务的对应关系：

| Agent 类型 | 对应的实际业务 | 实际操作示例 |
|-----------|--------------|------------|
| **AnalyzerAgent**（分析型） | 入场时的调研工作：读取门店数据、竞品数据、评价数据、ROS分状态，输出诊断报告 | 商圈竞对表制作、客流分析、经营评分诊断、门店竞争力分析 |
| **WebOperatorAgent**（电脑控制） | 在美团商户后台（网页端）执行操作：建团单、设推广、修改店铺信息、报名活动 | 创建团购套餐、调整代金券折扣、报名秒杀活动、修改门店营业时间 |
| **MobileOperatorAgent**（手机控制） | 在开店宝 App（手机端）执行操作：活动确认、素材检查、消息处理 | ROS分截图获取、查看门店营业状态、确认活动上线、检查商品图片 |
| **ReporterAgent**（汇报型） | 生成日报/周报，同步给商家和内部团队，通过企业微信群进行汇报 | 周数据报表、节假日活动复盘、ROS分变动通知、异常告警 |

## 六、加分题提示：前72小时运营逻辑

笔试加分题要求你设计新店前72小时的 Agent 执行优先级。以下是实际代运营中新店前几天的工作重点，供你参考思考：

- **第1天（入场当天）**：了解门店基础信息、查看销量/星级/评分/商品现状、与老板电话沟通确认店铺名称、数量、设计风格、营销活动意向
- **第2天（基础完善）**：核对套餐数量和活动、ROS分登记与完善、店铺装修（头图、套餐图）、确认设计风格
- **第3天（团单与营销）**：团购套餐变更与优化、设置好评/差评回复模板、官方相册完善、标签基础完善、根据店铺详情制定运营方案

> **思考提示**：新店样本中 `rating=4.0`, `gmv_last_7d=0`, `review_reply_rate=12%`, `ros_health=low`，说明这是一家刚开业、线上基础极差的新店。你需要思考：先做"地基"（基础信息+ROS分）还是先做"引流"（推广+活动）？在实际业务中，基础质量没做好就打曝光，转化率会很低，等于浪费钱。
